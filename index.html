Before You Start

  • Sign up for GitHub — Free at github.com/signup. You'll need this for hosting code and version control.
  • Git installed & configured — See our Git Setup Guide for reference. We'll walk through this together during office hours.
  • A terminal app — Mac: Terminal or iTerm2  |  Windows: PowerShell or CMD

Anthropic

Claude Code

Requires
No dependencies (native binary)
Account
Claude Pro, Max, Team, or Enterprise
Auto-updates
Yes (native install)
Platforms
macOS, Windows, Linux
Step 1

Install Claude Code

Run the installer for your OS. No Node.js needed — this is a native binary.

curl -fsSL https://claude.ai/install.sh | bash
irm https://claude.ai/install.ps1 | iex
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
Windows users: Install Git for Windows first. If your prompt shows PS C:\, you're in PowerShell — use the PowerShell command.
Step 2

Verify Installation

Open a new terminal window, then check the version:

claude --version
Step 3

Authenticate

Run claude and your browser will open for you to sign in with your Anthropic account.

claude
Tip: You can also run claude doctor to check that everything is configured correctly.
Step 4

Start Using It

Navigate to any project folder and run claude to start a conversation. Try something like:

cd my-project
claude
# Then type: "Explain this codebase to me"

Windows

WSL2 Setup (Optional)

What is it
Full Linux inside Windows
Requires
Windows 10 (1903+) or 11
Best for
Python-heavy workflows, Docker, bash scripts
Skip if
You just want to try the AI CLIs quickly
Do you need this? All three AI CLIs work natively on Windows with PowerShell. WSL2 is for Windows users who want a Linux environment — useful if you're running Python trading scripts, using Docker, or working with repos that assume bash. If you just want to try the tools, skip this and use the native Windows instructions.
Step 1

Install WSL2

Open PowerShell as Administrator (right-click → "Run as administrator") and run:

wsl --install

This installs WSL2 with Ubuntu by default. Restart your computer when prompted.

Tip: If you get an error, make sure virtualization is enabled in your BIOS. Most modern machines have it on by default.
Step 2

Set Up Ubuntu

After restart, Ubuntu will launch automatically (or search "Ubuntu" in Start). Create a username and password when prompted.

# You'll see something like:
Enter new UNIX username: yourname
New password: ********

Then update packages:

sudo apt update && sudo apt upgrade -y
Step 3

Install Node.js (for Gemini & Codex)

Use nvm to install Node.js — this avoids permission issues:

# Install nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash

# Reload shell
source ~/.bashrc

# Install latest LTS Node
nvm install --lts

# Verify
node --version   # Should show v20+
Step 4

Install the AI CLIs

Inside your WSL2 terminal, all three tools use their Linux install commands:

# Claude Code (native binary, no Node needed)
curl -fsSL https://claude.ai/install.sh | bash

# Gemini CLI
npm install -g @google/gemini-cli

# OpenAI Codex
npm install -g @openai/codex

Then authenticate each one the same way as the other tabs describe — run the command and follow the browser prompts.

Step 5

Access Your Windows Files

Your Windows drives are mounted under /mnt/:

# Your Windows home folder
cd /mnt/c/Users/YourName

# Desktop
cd /mnt/c/Users/YourName/Desktop

# A project on your D: drive
cd /mnt/d/projects/my-trading-bot
Tip: For best performance, keep project files inside the WSL filesystem (~/projects/) rather than on /mnt/c/. Cross-filesystem I/O is slower.
Step 6

Quick Reference

Handy commands for working with WSL2 day-to-day:

# Open WSL from PowerShell
wsl

# Open current Windows folder in WSL
wsl --cd .

# Open Windows Explorer from inside WSL
explorer.exe .

# Check your WSL version
wsl --list --verbose

# Shut down WSL (frees RAM)
wsl --shutdown

Common Issues

  • "command not found" — Close and reopen your terminal after installing. The PATH needs to refresh.
  • "permission denied" on Mac — Don't use sudo. If npm gives errors, fix npm permissions: npm config set prefix ~/.npm-global
  • Windows PowerShell won't run scripts — Run: Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
  • Node.js version too old — Check with node --version. Need 20+? Use nvm (Mac) or reinstall from nodejs.org
  • Claude Code says "not eligible" — Free Claude.ai accounts don't include Claude Code. You need Pro ($20/mo) or higher.
