Git Setup Guide

FF Office Hours — Install Git, configure SSH, and push your first commit

1

Install Git

Install

Download and install Git for your OS

# Option 1: Install via Homebrew (recommended)
brew install git

# Option 2: Install Xcode Command Line Tools
xcode-select --install

# Verify
git --version
# Download from https://git-scm.com/downloads/win
# Or install via winget:
winget install Git.Git
Installer settings we recommend:
• Default editor: Nano (avoid Vim if you're new to terminals)
• SSH: Bundled OpenSSH
• HTTPS backend: Native Windows Secure Channel
• Line endings: "Checkout as-is, commit Unix-style" (keeps LF everywhere)
• Terminal: MinTTY
• Git pull: Fast-forward or merge (default) 
# Ubuntu / Debian
sudo apt update && sudo apt install git -y

# Fedora
sudo dnf install git

# Arch
sudo pacman -S git

# Verify
git --version
# Git comes pre-installed in most WSL distros
# Update to the latest version:
sudo apt update && sudo apt install git -y

# Verify
git --version
Note: WSL2 has its own separate Git install from Windows Git. You'll want to configure both if you use both environments.
2

Configure Your Identity

Identity

Set your name and email

Every commit you make is tagged with your name and email. Set these globally so you don't have to repeat it per project.

git config --global user.name "Your Name"
git config --global user.email "your-email@example.com"
Use your GitHub no-reply email! If you use your personal email, GitHub may reject your push with the error: GH007: Your push would publish a private email address. To avoid this, use the no-reply email GitHub provides.
Email Privacy

Find your GitHub no-reply email

Go to github.com/settings/emails and look for an address like:

12345678+yourusername@users.noreply.github.com

Copy that address, then use it as your Git email:

git config --global user.email "12345678+yourusername@users.noreply.github.com"
Why this matters: Your commit email is public in every repo you push to. Using the no-reply address keeps your personal email private while still linking commits to your GitHub profile.
Defaults

Set recommended defaults

These settings will save you headaches down the road:

# Set default branch name to "main"
git config --global init.defaultBranch main

# Use Unix-style line endings (important if you also use WSL2)
git config --global core.autocrlf input

# Set nano as your default editor (avoid Vim)
git config --global core.editor "nano"

# Verify all your settings
git config --global --list

Set Up SSH Keys

Why SSH?

SSH vs HTTPS

There are two ways to authenticate with GitHub:

HTTPS — Simpler to start. Prompts you to sign in via browser when you push. Good for getting started quickly.

SSH — Requires one-time setup, but after that you never need to authenticate again. Recommended for regular use.

Recommendation: Set up SSH now. It takes 5 minutes and saves you from login prompts every time you push.
Step 1

Generate an SSH key

ssh-keygen -t ed25519 -C "your-email@example.com"

# Press Enter to accept default file location
# Enter a passphrase (optional — press Enter to skip)
# Run in PowerShell (not as admin)
ssh-keygen -t ed25519 -C "your-email@example.com"

# Press Enter to accept default file location
# Enter a passphrase (optional — press Enter to skip)

# Your key is saved at: C:\Users\YourName\.ssh\id_ed25519
ssh-keygen -t ed25519 -C "your-email@example.com"

# Press Enter to accept default file location
# Enter a passphrase (optional — press Enter to skip)

# Your key is saved at: ~/.ssh/id_ed25519
Step 2

Start the SSH agent and add your key

# Start the agent
eval "$(ssh-agent -s)"

# Add your key
ssh-add ~/.ssh/id_ed25519
# FIRST: Open PowerShell as Administrator and run these two lines:
Get-Service ssh-agent | Set-Service -StartupType Automatic
Start-Service ssh-agent

# THEN: In your regular (non-admin) PowerShell, add your key:
ssh-add ~\.ssh\id_ed25519
Common error: "Access is denied" on Set-Service means you're not in an admin PowerShell. Right-click PowerShell → "Run as administrator" for just those first two lines. 
# Start the agent
eval "$(ssh-agent -s)"

# Add your key
ssh-add ~/.ssh/id_ed25519
Step 3

Copy your public key

pbcopy < ~/.ssh/id_ed25519.pub
Get-Content ~\.ssh\id_ed25519.pub | Set-Clipboard
# If xclip is installed:
xclip -sel clipboard < ~/.ssh/id_ed25519.pub

# Otherwise, print it and copy manually:
cat ~/.ssh/id_ed25519.pub
Step 4

Add the key to GitHub

Your public key is now on your clipboard. Add it to your GitHub account:

1. Go to github.com/settings/keys
2. Click "New SSH key"
3. Title: something like "Windows PC" or "MacBook"
4. Paste your key (Ctrl+V / Cmd+V)
5. Click "Add SSH key"

Step 5

Test the connection

ssh -T git@github.com

You should see: Hi yourusername! You've successfully authenticated...

Tip: If it asks "Are you sure you want to continue connecting?" type yes. This adds GitHub to your known hosts list and only happens once.
4

Organize Your Repos

Folder Structure

Create a dedicated repos folder

Keep all your Git repositories in one place — not scattered across your Desktop or Downloads.

mkdir ~/repos
cd ~/repos
mkdir ~\repos
cd ~\repos
mkdir ~/repos
cd ~/repos
Windows users: Don't put repos inside OneDrive-synced folders! OneDrive can corrupt the .git directory and cause weird issues. The ~\repos folder sits outside OneDrive by default.
5

Your First Clone, Commit & Push

Clone

Clone a repository

Copy an existing GitHub repo to your machine. Always use the SSH URL (starts with git@github.com):

cd ~/repos
git clone git@github.com:username/repo-name.git
cd repo-name
Where to find the SSH URL: On the repo's GitHub page, click the green "Code" button, select the "SSH" tab, and copy the URL.
Edit

Make a change

Edit or create a file — anything will do for testing:

# Create a new file
echo "Hello from my first commit!" > test.txt
Stage

Stage your changes

Staging tells Git which changes you want to include in the next commit:

# See what changed
git status

# Stage a specific file
git add test.txt

# Or stage everything
git add .
Commit

Commit your changes

A commit is a snapshot of your staged changes with a descriptive message:

git commit -m "add test file"
Push

Push to GitHub

Send your commit to the remote repository on GitHub:

git push

If everything is set up correctly, this should complete with no prompts. Check your repo on GitHub — your changes should be there.

6

Essential Git Commands

Cheat Sheet

Commands you'll use every day

# Check what's changed
git status

# See the actual changes line by line
git diff

# View commit history
git log --oneline

# Pull latest changes from GitHub
git pull

# The full add → commit → push workflow
git add .
git commit -m "describe what you changed"
git push

# Create and switch to a new branch
git checkout -b branch-name

# Switch back to main
git checkout main

# Undo changes to a file (before staging)
git checkout -- filename

# Unstage a file (after git add, before commit)
git reset HEAD filename

Troubleshooting

Permission denied (publickey)
Your SSH key isn't set up or isn't being found. Make sure you've added it to the SSH agent and to your GitHub account. Run ssh -T git@github.com to test. If it fails, re-do Phase 3.
GH007: Your push would publish a private email address
You're using your personal email in commits and GitHub's email privacy is blocking it. Fix by switching to your no-reply email: 
git config --global user.email "ID+user@users.noreply.github.com"
git commit --amend --reset-author --no-edit
git push
fatal: Unable to create '.../.git/index.lock': File exists
A previous Git operation crashed and left a lock file behind. Delete it and retry: 
# macOS / Linux / WSL2
rm .git/index.lock

# Windows PowerShell
Remove-Item .git\index.lock
Access is denied — Set-Service ssh-agent
The SSH agent service commands need admin privileges on Windows. Right-click PowerShell → "Run as administrator", run the two service commands, then go back to your regular PowerShell for everything else.
PowerShell: "cannot be loaded because running scripts is disabled"
Windows blocks scripts by default. Run this once in PowerShell: 
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
"command not found" after installing Git
Close your terminal completely and open a new one. The PATH environment variable needs to reload for the system to find Git.
Line ending warnings (LF will be replaced by CRLF)
This happens when Windows and Linux line endings mix. Set this to normalize everything: 
git config --global core.autocrlf input

Setup Checklist

Git installed and git --version works
Name set with git config --global user.name
GitHub no-reply email set with git config --global user.email
Default branch set to main
Line endings set to core.autocrlf input
Default editor set to nano
SSH key generated (~/.ssh/id_ed25519 exists)
SSH key added to SSH agent
SSH public key added to GitHub settings
ssh -T git@github.com says "successfully authenticated"
Repos folder created (~/repos)
Successfully cloned, committed, and pushed to a repo
