Stock Monitor Demo

An AI-Native workflow for building a financial data dashboard

Project Goal

Build a web app that tracks stock tickers and displays: 2-year High/Low offsets, P/E & PEG ratios, Price-to-Sales, and a Momentum indicator (Trending Up/Down/Consolidating).

1

Planning Mode

Start by asking the AI to architect the system. Do not write code yet.

The Prompt I want to build a Stock Monitor web app. - Features: Ticker input, high/low offsets, financial ratios, momentum indicator. - Tech: Node.js/Express backend, Vanilla JS frontend. - API: Use Alpha Vantage or Yahoo Finance. Please enter PLANNING MODE and draft a technical architecture. Focus on data flow, caching to avoid API rate limits, and how momentum will be calculated.
2

The Deep-Dive Interview

Now, let the AI probe for missing requirements. This prevents "guesswork" coding.

The Prompt Based on that plan, please ask me 3-5 high-impact clarifying questions. Think like a Senior Engineer trying to avoid edge-case failures (e.g., market-closed data, invalid tickers, API timeouts).
Mentor Tip: Pay attention to the "Momentum" definition. Is it a 50-day moving average or simple 5-day trend? Get specific here!
3

Senior Dev Mentor (Plan Review)

Invoke a specialized "Mentor Agent" to critique the final architecture before a single line is written.

The Prompt Act as a Senior Staff Engineer. Review the current plan. Identify 3 potential architectural risks (e.g., race conditions on ticker updates, missing error boundaries, or improper data sanitization) and suggest improvements.
4

Implementation & Docs

Execute the plan. Have the AI write the code and the internal documentation simultaneously.

The Prompt Now, implement the project following the approved plan. 1. Create the server-side logic first. 2. Write a README.md explaining the "Why" behind the momentum calculation. 3. Build the frontend UI.
5

Testing & Security Audit

Don't trust the first output. Have a QA agent verify it and a Security agent scan it.

The Prompt 1. Act as a QA Engineer: Write and run tests that mock the stock API. Ensure the 'Consolidation' state is triggered if price moves < 1% over 5 days. 2. Act as a Security Auditor: Scan the code for hardcoded API keys or sensitive data exposure.
6

The Senior Dev PR Review

The final step. Treat the finished code like a Pull Request at a top-tier tech company.

The Prompt Perform a final Pull Request review. Critique code readability, use of modern JS features (like AbortController for fetch), and whether the financial math is robust. Suggest 2 final refactors for performance.
